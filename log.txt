//git 命令流程：
//a.同步代码
1.git branch(查看所在分支)
2.git checkout master(如果不在master分支，请切换到master主干)
3.git pull -v upstream master(从上游的master主干上同步更新代码)
4.git push -v origin master(更新代码到自己远程的master主干上)
(3,4 == git remote update upstream)
5.git checkout branch-name(切换到分支)
5.git merge master(合并master上的代码到当前分支)
//b.提交代码
git status(查看代码改动)
git add ./*(所有改动代码放到暂存区(Stage))
git commit -m ""(提交代码到本地分支上)
git checkout master(切换到master主干上)
git merge <branch name>(合并<branch name>到master上)
git pull
git push

多人协作的工作模式通常是这样：
首先，可以试图用git push origin branch-name推送自己的修改；
如果推送失败，则因为远程分支比你的本地更新，需要先用git pull试图合并；
如果合并有冲突，则解决冲突，并在本地提交；
没有冲突或者解决掉冲突后，再用git push origin branch-name推送就能成功！
